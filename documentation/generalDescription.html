1. Endpoints de Usuário

/api/users/register

Método: POST

Descrição: Registra um novo usuário.

Parâmetros de Requisição:

Parâmetro Tipo Descrição Obrigatório
username string Nome de usuário Sim
password string Senha do usuário Sim

Exemplo de Requisição:

POST /api/users/register
{
    "username": "novoUsuario",
    "password": "senhaSegura"
}

Resposta de Sucesso:

{
    "username": "novoUsuario",
    "message": "Usuário registrado com sucesso"
}

Resposta de Erro:

{
    "error": "Nome de usuário já está em uso"
}

/api/users/login

Método: POST

Descrição: Realiza o login do usuário.

Parâmetros de Requisição:

Parâmetro Tipo Descrição Obrigatório
username string Nome de usuário Sim
password string Senha do usuário Sim

Exemplo de Requisição:

POST /api/users/login
{
    "username": "usuarioExemplo",
    "password": "senhaSegura"
}

Resposta de Sucesso:

{
    "message": "Login realizado com sucesso"
}

Resposta de Erro:

{
    "error": "Credenciais inválidas"
}

/api/users/logout

Método: POST

Descrição: Realiza o logout do usuário, invalidando a sessão.

Exemplo de Requisição:

POST /api/users/logout

Resposta de Sucesso:

{
    "message": "Logout realizado com sucesso"
}

Resposta de Erro:

{
    "error": "Sessão expirada ou usuário não autenticado"
}

/api/users/user-info

Método: GET

Descrição: Retorna as informações do usuário logado, incluindo o saldo.

Resposta de Sucesso:

{
    "username": "usuarioExemplo",
    "balance": 100
}

Resposta de Erro:

{
    "error": "Usuário não autenticado"
}

/api/users/update-user-balance

Método: POST

Descrição: Atualiza o saldo de paint_drops do usuário.

Parâmetros de Requisição:

Parâmetro Tipo Descrição Obrigatório
credits number Quantidade de paint_drops a ser adicionada ou subtraída Sim

Exemplo de Requisição:

POST /api/users/update-user-balance
{
    "credits": 50
}

Resposta de Sucesso:

{
    "username": "usuarioExemplo",
    "newBalance": 150,
    "message": "Saldo atualizado com sucesso"
}

Resposta de Erro:

{
    "error": "Saldo insuficiente"
}

2. Endpoints de Jogos

/api/games/play

Método: POST

Descrição: Inicia um novo jogo (por exemplo, Blackjack ou Slot Machine).

Parâmetros de Requisição:

Parâmetro Tipo Descrição Obrigatório
gameType string Tipo de jogo (ex: blackjack, slot-machine) Sim
betAmount number Valor da aposta Sim

Exemplo de Requisição:

POST /api/games/play
{
    "gameType": "blackjack",
    "betAmount": 100
}

Resposta de Sucesso:

{
    "gameId": 123,
    "message": "Jogo iniciado com sucesso"
}

Resposta de Erro:

{
    "error": "Saldo insuficiente"
}
